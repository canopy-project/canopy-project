Cloud Variables

Canopy takes a unique and extensible approach to storing device data in the cloud. Our approach is based on a special type of cloud resource called a Cloud Variable.

A Cloud Variable is a piece of data that is stored in the cloud and tracked over time.

Canopy clients keep a cached local copy of each Cloud Variable and can synchronize with the cloud. Similar to the way GIT can synchronize local and remote copies of files, Canopy can synchronize local and remote copies of data.

Declaring a Cloud Variable

Before using a Cloud Variable you must declare it, specifying its name, direction, datatype and other metadata properties
